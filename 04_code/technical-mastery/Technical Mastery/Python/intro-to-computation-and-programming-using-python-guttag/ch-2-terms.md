### ✅ Programming & Language Basics

- **Low-level language**: A programming language closer to machine code (e.g., Assembly); offers direct control over hardware.
    
- **High-level language**: Closer to human language, abstracting hardware details (e.g., Python, Java).
    
- **Interpreted language**: Code is executed line-by-line by an interpreter (e.g., Python).
    
- **Compiled language**: Code is translated into machine code before execution (e.g., C, C++).
    
- **Source code**: Human-readable code written in a programming language.
    
- **Machine code**: Binary code executed directly by the computer's CPU.
    

---

### ✅ Python-Specific Terms

- **Python**: A high-level, interpreted language known for readability and versatility in data science, web dev, automation, and more.
    
- **IDE (Integrated Development Environment)**: A software suite that combines code editing, debugging, and execution tools (e.g., Spyder, VS Code).
    
- **Anaconda**: A Python distribution for data science that includes packages, Python, Jupyter, and tools like Spyder.
    
- **Spyder**: A Python IDE tailored for scientific programming, included with Anaconda.
    
- **IPython console**: An enhanced Python interpreter providing rich introspection, syntax highlighting, and ==magic commands==.
    
- **Shell**: A command-line interface to interact with the OS or interpreter.
    
- **Shell prompt**: The line where you type commands in a shell (e.g., `>>>` in Python).
    

---

### ✅ Programming Concepts

- **Program (script)**: A file containing Python code to be executed.
    
- **Command (statement)**: A single instruction (e.g., a print statement).
    
- ==**Object**: Any data structure or value in Python; everything is an object.==
    
- **Type**: Classification of data (e.g., `int`, `str`, `list`).
    
- **Scalar object**: Represents a single value (e.g., `int`, `float`).
    
- **Non-scalar object**: Holds multiple values (e.g., lists, dictionaries).
    
- **Literal**: A fixed value directly used in code (e.g., `42`, `'hello'`).
    
- **Floating point**: Numbers with decimals (e.g., `3.14`).
    
- **bool**: Boolean data type, `True` or `False`.
    
- **None**: A special constant in Python representing “no value”.
    

---

### ✅ Logic & Syntax

- **Operator**: Symbols used for operations (`+`, `-`, `*`, etc.).
    
- **Expression**: A combination of values and operators that evaluates to a result.
    
- **Value**: The data that an expression evaluates to.
    
- **Variable**: A name bound to a value in memory.
    
- **Binding**: Associating a variable with a value.
    
- **Assignment**: Giving a variable a value using `=`.
    
- **Reserved word**: Words Python uses for its syntax (e.g., `if`, `def`)—can’t be used as variable names.
    
- **Comment (in code)**: Non-executable text for documentation, marked with `#`.
    

---

### ✅ Control Structures

- **Straight-line program**: Executes code from top to bottom without branches.
    
- **Branching program**: Uses conditionals (`if`, `else`) to choose execution paths.
    
- **Conditional**: Statements that run based on whether an expression is true.
    
- **Indentation (in Python)**: Used to define code blocks—essential for structure.
    
- **Nested statement**: A control structure placed within another.
    
- **Compound expression**: Combines multiple expressions using logic (e.g., `a and b`).
    
- **Conditional expression**: A compact `if` form (e.g., `x if a > b else y`).
    

---

### ✅ Performance

- **Constant time**: An operation that completes in the same amount of time regardless of input size.
    
- **Computational complexity**: Describes how resource use scales with input size.
    

---

### ✅ Data Handling

- **Strings**: Textual data enclosed in quotes (e.g., `"hello"`).
    
- **Overloaded operator**: Operators that behave differently depending on operand types (`+` adds numbers, concatenates strings).
    
- **Repetition operator**: `*` when used with strings/lists (e.g., `"ha" * 3` → `"hahaha"`).
    
- **Type checking**: Determining a variable’s type, often using `type()` or `isinstance()`.
    
- **Indexing**: Accessing individual elements (e.g., `my_list[0]`).
    
- **Slicing**: Getting a range from sequences (e.g., `my_list[1:3]`).
    
- **Type conversion (casting)**: Changing one type to another (e.g., `int("5")`).
    
- **Formatted string expression**: Embedding values inside strings using `f""` (e.g., `f"Hello {name}"`).
    
- **Input**: Gathering user input via `input()`.
    

---

### ✅ Character Sets & Loops

- **Unicode**: International character encoding standard; Python strings are Unicode by default.
    
- **Iteration (looping)**: Repeating code, typically with `for` or `while`.
    
- **Pseudocode**: Plain language used to outline code logic without syntax.
    
- **while loop**: Repeats while a condition is `True`.
    
- **for loop**: Iterates over a sequence (e.g., `for x in range(5):`).
    
- **break**: Exits a loop early.
    
- **hand simulation**: Manually walking through code to understand behavior.
    

---

### ✅ Advanced Python

- **tuple**: Immutable sequence of items (e.g., `(1, 2, 3)`).
    
- **range**: A built-in function for sequences of numbers (e.g., `range(5)`).
    
- **in operator**: Checks if a value exists in a sequence (`5 in [1, 2, 5]` → `True`).
    
- **PEP 8 style guide**: Python’s official style guidelines, covering naming, spacing, and best practices.
